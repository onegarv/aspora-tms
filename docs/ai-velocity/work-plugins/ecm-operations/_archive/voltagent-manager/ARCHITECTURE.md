## MCP Protocol

ecm-gateway uses JSON-RPC 2.0 over HTTP POST:

```bash
curl -X POST 'http://ecm-gateway:80/mcp' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","method":"tools/call","params":{"name":"redshift_execute_sql_tool","arguments":{"sql":"SELECT..."}},"id":1}'
```

Available tools:
- `redshift_execute_sql_tool` â€” Run read-only SQL
- `sheets_get_sheet_data` â€” Read Google Sheet
- `sheets_update_cells` â€” Write to Google Sheet

## Why This Architecture?

| Old Approach | New Approach |
|--------------|--------------|
| TypeScript parses SQL from skills | LLM reads skill, calls MCP tools |
| Hardcoded data fetching logic | Skill defines what data to fetch |
| Executor generates briefing | LLM generates briefing |
| Business logic in code | Business logic in skills |
| 500+ lines of TypeScript | 312 lines total |

**The skill IS the program. TypeScript is just the trigger.**
